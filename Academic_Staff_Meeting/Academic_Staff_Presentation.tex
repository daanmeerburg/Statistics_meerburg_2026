\begin{frame}{What I Built: Executable Course Material}
  \begin{columns}[T,totalwidth=\textwidth]
    \begin{column}{0.52\textwidth}
      \begin{itemize}
        \item New statistics course for third-year BSc Physics students
        \item 12 Jupyter lecture notebooks: text, equations, code, figures, exercises
        \item Students run the same file they read in class
        \item Colab for immediate access; VS Code/Git for persistent local editing
        \item Board notes, quizzes, assignments, projects, exams, rubrics
        \item HPC development workflow synchronized with Git and Overleaf
      \end{itemize}
    \end{column}
    \begin{column}{0.44\textwidth}
      \begin{block}{Central choice}
        \Large Jupyter notebooks
      \end{block}
      \begin{block}{Design principle}
        Keep statistical theory, Python implementation, output, and interpretation together.
      \end{block}
      \begin{block}{In class}
        Students can rerun and modify examples while the statistical idea is discussed.
      \end{block}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Two options for students to run notebooks}
  \begin{columns}[T,totalwidth=\textwidth]
    \begin{column}{0.48\textwidth}
      \begin{block}{Option 1: Colab}
        \begin{itemize}
          \item click the Colab badge
          \item run cells immediately
          \item many common Python libraries already available
          \item use Drive/download/GitHub export for lasting copies
        \end{itemize}
      \end{block}
    \end{column}
    \begin{column}{0.48\textwidth}
      \begin{block}{Option 2: VS Code + Git}
        \begin{itemize}
          \item Fork or clone the repository
          \item Edit notebooks locally
          \item Commit and maintain a personal version
          \item More package setup, more flexibility
        \end{itemize}
      \end{block}
    \end{column}
  \end{columns}
  \vspace{0.35cm}
  \begin{block}{Trade-off}
    Colab is easiest for class use, but runtime state is temporary. VS Code and Git require local setup, but give students a persistent fork they can edit and maintain.
  \end{block}
\end{frame}

\begin{frame}{Student Projects}
  \small
  \begin{columns}[T,totalwidth=\textwidth]
    \begin{column}{0.52\textwidth}
      \begin{block}{Project structure}
        \begin{itemize}
          \item Timeline: 6 lecture weeks, then 2 project weeks
          \item Max. 5 students per project
          \item Student-facing project template, refined with AI support
          \item Notebook analysis plus final report
          \item Aim: a small research-style statistics analysis
        \end{itemize}
      \end{block}
    \end{column}
    \begin{column}{0.44\textwidth}
      \begin{block}{Git and assessment}
        \begin{itemize}
          \item Students push notebooks and reports to Git
          \item Pushes and pull requests make contributions visible
          \item Report piloted as pass/fail
          \item Exam includes one project question: parts (a)--(d)
          \item (a,b): report literacy; (c,d): understanding
        \end{itemize}
      \end{block}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Things to keep in mind}
  \begin{columns}[T,totalwidth=\textwidth]
    \begin{column}{0.49\textwidth}
      \begin{block}{Challenges}
        \begin{itemize}
          \item Colab sessions are convenient but temporary.
          \item VS Code/Git gives flexibility, but needs setup.
          \item Notebooks can hide complexity.
          \item AI can produce plausible errors.
        \end{itemize}
      \end{block}
    \end{column}
    \begin{column}{0.49\textwidth}
      \begin{block}{Mitigations}
        \begin{itemize}
          \item Explicit persistence/forking instructions (lecture 1)
          \item During lecture I combine blackboard presentation with live notebook execution
          \item Students are encouraged to read and understand the code, not just run it
          \item AI is used as a drafting tool, but all changes are reviewed before release
          \item Private repos for exams and solutions, public repos for lectures and assignments
        \end{itemize}
      \end{block}
    \end{column}
  \end{columns}
\end{frame}

\begin{frame}{Take-Home Message}
  \begin{center}
    \Large
    The course is built around version-controlled Jupyter notebooks. It uses many modern tools to make the material interactive, reproducible, and maintainable. AI integration supports this approach. The course is still in its first year, but the design principles and workflow are already showing benefits for students and for me as an instructor.
  \end{center}
  \vspace{0.4cm}
  \begin{itemize}
    \item Students can read, run, fork, and modify the lectures.
    \item They can choose Colab for access or VS Code/Git for persistence.
    \item Git, Overleaf, and the HPC make the material easier to maintain.
    \item AI helps with drafting and repository maintenance, but changes are reviewed before release.
  \end{itemize}
\end{frame}
